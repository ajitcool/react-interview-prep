# HR Interview Questions and Answers

## 1. Tell Me About Yourself?
I'm a Frontend Developer with 5 years of experience, primarily in React.js ecosystem. My responsibilities include designing and building scalable component architecture, leading frontend development for key modules, mentoring junior developers, and conducting code reviews. I work closely with backend teams for API design, with UX for pixel-perfect implementations. I've also handled performance optimization —  like lazy loading, code splitting, and bundle size reduction — and have contributed to tech stack decisions and frontend best practices in my team.

## 2. What Are Your Strengths & Weaknesses?
**Strength 1 — Problem Solving:** "Main complex React performance issues quickly identify kar leta hun. Ek project mein maine render time 3 seconds se 0.4 seconds tak laya sirf memoization aur code splitting se."

**Strength 2 — Ownership:** "Main sirf assigned kaam nahi karta — agar koi bug ya better approach dikhti hai, main proactively fix karta hun bina kisi ke bolne ka wait kiye."

**Strength 3 — Quick Learner:** "Maine 5 saal mein khud ko functional compoenent se Hooks tak, JavaScript se TypeScript tak continuously upgrade kiya."

Meri weakness yeh thi ki main pehle documentation likhna avoid karta tha — mujhe lagta tha code self-explanatory hai. Lekin jab team mein junior developers aaye, tab samjha ki documentation kitni zaroori hai. Ab main har feature ke saath README aur JSDoc comments likhta hun.

## 3. Where Do You See Yourself in 5 Years?
Agle 5 saal mein main ek strong senior React developer banna chahta hun — jahan main architecture decisions mein actively contribute karun. Agle 5 saal mein main ek frontend lead ya tech lead role mein rehna chahta hun — jahan main ek team guide karun, technical decisions lun, aur product ki growth mein directly responsible rahun. Aur honestly — main ek aisi company mein yeh journey karna chahta hun jahan product matter karta ho. Isliye main aapki company mein interested hun.

## 4. Why Should We Hire You?
"Teen reasons hain: Pehla — Main production-ready code likhta hun. 5 saal mein maine real users ke liye real applications banaye hain — sirf tutorials nahi kiye.

Doosra — Main problems proactively dhundta hun. Ek baar hamare app ki performance suddenly drop hui — maine khud React Profiler se root cause dhundha aur 2 din mein fix kiya, bina kisi ke bolne ka wait kiye.
 
Teesra — Main team player hun. Maine TCS mein junior developers ko informally mentor kiya hai — code reviews, pair programming, aur knowledge sharing sessions. In teeno cheezoon ka combination — technical skills, ownership, aur team mindset — yahi mujhe valuable banata hai."

## 5. What Is Your Biggest Achievement?
"Mera sabse bada achievement ek dashboard project tha jisme 500+ rows ki table thi. Problem yeh thi ki har filter change pe poori table re-render hoti thi — app almost unusable tha slow devices pe. Maine React DevTools Profiler se root cause identify kiya, React.memo, useCallback, aur virtual scrolling implement kiya. 
Result: Render time 4 seconds se 0.3 seconds. User complaints zero. Client ne specifically is improvement ko appreciate kiya. Yeh mera biggest achievement isliye hai kyunki maine khud initiative liya, technically solve kiya, aur real business impact hua."

## 6. How Do You Handle Pressure & Conflict
**Pressure:** "Pressure mein main pehle problem ko prioritize karta hun — kya most critical hai, pehle woh fix karo. Ek baar production mein bug aaya Friday evening ko — deploy ke baad. Main ghabraya nahi, quietly error logs dekhe, root cause dhundha, hotfix push kiya — 2 ghante mein resolved. Pressure mein calm rehna aur systematically sochna — yeh maine TCS mein seekha."

**Conflict:** "Conflict hone pe main pehle dusre ki baat poori sunata hun. Ek baar ek senior developer se architecture ko lekar disagreement tha — woh Redux chahte the, main React Query. Maine dono approaches ka pros/cons document banaya, team ko present kiya — data se decision liya, ego se nahi. Ultimately React Query choose hua aur project successful raha."

## 7. What Is Your Notice Period?
"Mera official notice period TCS mein 2 months ka hai. Lekin agar aapko jaldi joining chahiye, main HR se baat karke early relieving ki request kar sakta hun. Aur agar aapki joining flexible hai, main poora transition properly complete karke aana chahta hun — yeh mere professionalism ka hissa hai."

## 8. Expected Salary Negotiation Script
**Jab Poochhe "What Are Your Expectations?"** "Based on my 5 years of hands-on experience with React, TypeScript, and Redux, aur current market rate ko consider karte hue, main 12 to 15 LPA expect kar raha hun. Lekin main overall package ko consider karta hun — growth opportunity, learning, aur role responsibility bhi equally important hai mere liye."

**Agar Kam Offer Aaye:** "Main aapka offer appreciate karta hun. Meri research aur experience ke basis pe main 18 LPA expect kar raha tha. Kya is range mein koi flexibility hai? Main genuinely is role mein interested hun — isliye pooch raha hun."
